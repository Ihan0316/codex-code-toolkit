워크스페이스 전체의 자동 일간보고 루틴. 모든 출력은 한국어, 사실 기반·간결.
**코드/외부 산출물 자동 수정 금지(읽기·분석만)** — 변경은 제안만, 실제 적용은 사용자 확인 하에.

> Codex 스킬 배치 위치: 사용자 `~/.agents/skills/daily-report/` 또는 프로젝트 `.agents/skills/daily-report/`.
> 헤드리스로 돌리려면 `codex exec --sandbox read-only "일일보고 작성"` (04-automation 참고).

## 경로 자동 탐지 (하드코딩 금지)

- **워크스페이스 루트** = cwd에서 상위로 올라가 `<WORKSPACE>` 폴더 (없으면 cwd를 루트로 가정).
  - macOS/Linux: `~/Documents/<WORKSPACE>/`  (Windows는 WSL2 안의 동일 경로)
- **오늘 날짜** = 시스템 현재 날짜(YYYY-MM-DD). 주차 = "N월 N주차"(월요일 시작).
- **저장 루트**: `<워크스페이스 루트>/<일일보고 폴더>/<N월>/<N주차>/<오늘>.md`

## 프로젝트 정의 (자동 화이트리스트)

- **프로젝트 = 워크스페이스 루트 직하위 폴더 중 `AGENTS.md`가 있는 폴더.**
- 새 프로젝트를 보고 대상에 넣으려면 그 폴더에 `AGENTS.md`만 두면 됨(등록 불필요).

## 절차

1. 오늘 각 프로젝트의 `git log`·변경 파일·작업 내역을 읽어 **사실 기반**으로 요약.
2. 프로젝트별 섹션으로 일일보고 md 작성/갱신(같은 날 여러 프로젝트면 섹션 누적, 동일 프로젝트 섹션만 치환).
3. (선택) 외부 보고 DB(노션 등, MCP 연결 시)에 동일 내용 기록.
4. 로드맵/진척 대비 편차가 있으면 **보고만**(자동 수정 금지).

> 외부 시스템 기록 생성은 사용자 정책에 따라. 결제·발송류는 항상 확인.
